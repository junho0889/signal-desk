# Raspberry Pi Collector Node

## Purpose
Provide a dedicated always-on collection node using Raspberry Pi 4B 8GB, separate from the central database and API host.

## Frozen Role Boundary (COL-001)
- run source adapters 24/7 for `news_primary`, `search_trends`, `market_ohlcv`, and `dart_disclosures`
- persist raw payloads to local spool before any central transfer attempt
- retry central transfers with bounded backoff and dead-letter handling
- expose health, backlog, and last-success telemetry for operations
- never become the authoritative ranking, API, or PostgreSQL host

## Recommended Node Profile
- Raspberry Pi 4B 8GB
- SSD-backed storage preferred over SD-card-only operation
- wired Ethernet preferred
- UPS or equivalent power resilience preferred for overnight collection

## Explicit Non-Goals
- do not run the primary PostgreSQL instance on the Pi
- do not run heavy ranking/model workloads on the Pi
- do not make the Pi the only copy of raw collected payloads

## Collector Runtime Topology
1. scheduler
- triggers source adapter cycles using source-specific cadence from `docs/data/source-catalog.md`

2. adapters
- poll or intake source payloads and validate required source fields
- emit immutable payload + envelope for spool writer

3. spool writer
- writes each payload durably to local storage with `status=queued`
- collection cycle is considered successful only after spool write succeeds

4. transfer worker
- leases queued items, attempts central delivery, and records acknowledgement metadata
- applies retry/backoff policy and dead-letter transitions

5. prune worker
- removes only `acked` and expired `dead_letter` records by retention policy

6. health emitter
- reports queue depth, oldest unshipped age, dead-letter count, per-source success/failure rate, and last successful ack time

## Delivery Modes To Support
- direct central intake API
- file or object upload into a shared intake location
- deferred resend when central network is unavailable

Delivery transport is pluggable; acknowledgement semantics are fixed.

## Local Spool Contract (Frozen)
Each spool item contains immutable payload content and mutable transfer metadata.

### Required Metadata
| field | requirement |
|---|---|
| `spool_id` | unique UUID generated by collector |
| `source_id` | one of `news_primary`, `search_trends`, `market_ohlcv`, `dart_disclosures` |
| `adapter_mode` | `poll`, `webhook`, or `backfill` |
| `collected_at_utc` | required UTC timestamp |
| `upstream_event_ts_utc` | optional UTC event timestamp from source |
| `idempotency_key` | deterministic key using source-catalog identity rules |
| `payload_hash_sha256` | required content hash |
| `payload_encoding` | `json`, `json_gzip`, or `csv` |
| `payload_bytes` | payload size in bytes |
| `status` | `queued`, `inflight`, `acked`, `dead_letter` |
| `retry_count` | transfer retry attempts |
| `next_attempt_at_utc` | scheduler hint for next transfer attempt |
| `ack_code` / `ack_id` / `acked_at_utc` | populated only after central response |
| `last_error_code` / `last_error_at_utc` | populated on failed attempts |
| `expires_at_utc` | retention deadline |

### Default On-Disk Layout
- root: `/var/lib/signaldesk/spool/`
- queues:
  - `queued/<source_id>/YYYY/MM/DD/`
  - `inflight/<source_id>/YYYY/MM/DD/`
  - `acked/<source_id>/YYYY/MM/DD/`
  - `dead-letter/<source_id>/YYYY/MM/DD/`
- each spool item stores:
  - metadata record (json)
  - raw payload file (content-addressed by `payload_hash_sha256` when feasible)

Implementations may vary physically, but they must preserve the logical states and metadata fields.

## Source Adapter Expectations (Frozen)
| source_id | cadence | required source fields | notes |
|---|---|---|---|
| `news_primary` | 30 min | `external_id`, `published_at`, `source_name`, `title`, `url`, `language`, `raw_text_or_summary` | dedupe later in central normalization, not on Pi |
| `search_trends` | 60 min | `keyword`, `window_start`, `window_end`, `interest_index`, `region`, `sample_granularity` | sampled data is accepted; freshness flags handled centrally |
| `market_ohlcv` | 15 min during market sessions + daily close backfill | `symbol`, `ts`, `open`, `high`, `low`, `close`, `volume`, `market` | partial market outages should not block other adapters |
| `dart_disclosures` | 30 min | `filing_id`, `filed_at`, `issuer`, `symbol`, `filing_type`, `title`, `url` | issuer/symbol ambiguity resolved centrally |

## Retry And Delivery Policy (Frozen)
- source fetch retries: up to 5 attempts per cycle with exponential backoff
- central delivery retries:
  - retryable: timeout, network failure, 429, 5xx, temporary intake unavailable
  - non-retryable: contract rejection, malformed payload, unauthorized credentials
  - schedule: 30s base, multiplier 2.0, jitter +/-20%, max interval 10m
  - cutoff: 20 attempts or 24h age, then `dead_letter`

## Central Acknowledgement Contract (Frozen)
Central intake must return:
- `ack_code`: `accepted`, `duplicate`, or `rejected`
- `ack_id`: unique central acknowledgement id
- `acked_at_utc`: central handling timestamp
- optional `reason_code` on rejection

Collector handling:
- `accepted` -> mark `acked`
- `duplicate` -> mark `acked` (already durable centrally)
- `rejected` -> mark `dead_letter` with rejection reason
- no response/timeout -> remain or return to `queued` for retry

## Retention And Prune Rules
- keep `acked` items for at least 24h before prune
- keep `dead_letter` items for 7 days before prune
- do not prune `queued` or `inflight`
- when spool usage exceeds 80%, disable non-critical backfill work and alert operator

## Operational Signals
- spool directory size
- oldest unshipped payload age
- source success rate
- source error rate
- last successful ship timestamp
- dead-letter queue count and oldest dead-letter age
- per-source retry budget exhaustion count

## Security Notes
- Pi holds only the minimum credentials required to ship payloads centrally
- central authority remains on the main host
- secrets must stay outside version control
- acknowledgement identity should be logged with enough metadata for replay audit
