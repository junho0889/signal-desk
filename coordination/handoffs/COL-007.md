## Root Cause Isolation
- network is reachable:
  - `Test-NetConnection 192.168.0.33 -Port 22` -> `TcpTestSucceeded : True`
- SSH server is reachable and offers auth methods:
  - `ssh -vvv -o BatchMode=yes -o ConnectTimeout=10 -p 22 admin@192.168.0.33 "echo SSH_OK"`
  - key evidence in output:
    - `Authentications that can continue: publickey,password`
    - default identities not found (`id_rsa`, `id_ed25519`, etc.)
    - `No more authentication methods to try.`
- local SSH credential state:
  - `Get-ChildItem $env:USERPROFILE\.ssh -Force` -> only `known_hosts`
  - no local private key available for non-interactive login

## Focused Fix Applied
- added deterministic Pi smoke script:
  - `infra/collector/pi-remote-smoke.ps1`
  - enforces key-based SSH preflight and runs the remote boot + ingest + spool-evidence sequence when auth is valid
- updated deployment runbook with collector Pi auth gate and recovery path:
  - `docs/ops/deploy-runbook.md`
  - added exact preflight command, root-cause explanation, and remote smoke command surface
- imported collector infra assets required by the deployment path:
  - `infra/collector/`
